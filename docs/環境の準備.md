# 環境の準備

pythonを実行するための環境を作ります

この引き継ぎでは一連のコードの実行を`.ipynb`ファイルで行います。

本格的に大規模な機械学習を行う時は、研究室2337のマシン`ryokucha`を使用します。`ryokucha`は現在空き容量不足のためHDDを増設する必要があり、アカウント作成はHDD増設後になるため、しばらくの間はローカル環境で実行することになります(環境が用意できない場合はGoogleColaboratoryを使用します)。

すでにPCにpythonの実行環境がある場合はそれを使って問題ありません。

2024.11.25現在、`Python 3.13`が最新版として使用可能になっていますが、主要な機械学習ライブラリ等の対応がまだ行われていないため、`3.11`か`3.12`を使用してください。

もしすでにある実行環境が`Python 3.13`以降の場合はpyenv等でバージョンを切り替えて`3.11`か`3.12`を使用してください

OSにインストールされているpythonに影響を与えないために、ここでは仮想環境(venv)を作る方法の説明をします

---

### venvとは

Pythonで仮想環境を作成するためのツールです。プロジェクトごとに独立したPython環境を作ることで、異なるプロジェクトで使用するライブラリのバージョンや設定を分離できます。これにより、1つの環境を汚すことなく、異なるPythonプロジェクトを同じマシン上で動かせます。
venvでインストールしたライブラリは他のプロジェクトに影響しません。

### 作り方

VSCodeで拡張機能`Python(Microsoft)`と`Jupyter(Microsoft)`をインストールします

venvはVSCodeから`カーネルの選択`>`Python環境`>`Pyton環境の作成`>`venv`で作成することもできます



以下、コマンドを実行してvenvを作成する方法を説明します。(コマンドプロンプトやターミナルを使用します)

pyvenvという名前の仮想環境を作る場合は以下のようにコマンドを実行します
```
python3 -m venv pyvenv
```
または
```
python -m venv pyvenv
```

`pyvenv`というフォルダが作成されます

仮想環境を有効化します

mac/linuxの場合
```
source pyvenv/bin/activate
```

windowsの場合
```
myenv\Scripts\activate
```

成功するとプロンプトに仮想環境名が表示されます
```
(pyvenv) $
```

仮想環境を終了するには次のコマンドを実行します
```
deactivate
```

### ipynbから使用する方法

VSCodeで`.ipynb`ファイルを開いて、`カーネルの選択`の一覧から`pyvenv`を選択します

これで`.ipynb`でpythonが実行可能になりました
